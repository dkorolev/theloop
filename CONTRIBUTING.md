# Contributing to theloop

This file states the principles *beneath* the repository's mechanical rules — the why, for humans and agents deciding things the rules don't cover. It deliberately does not restate what is already codified elsewhere (that would violate principle 1). The codified law lives in:

* [`.theloop/SKILLS-META-RULES.md`](.theloop/SKILLS-META-RULES.md) — how skills are authored: run receipts, naming, scripts, containment, parallel fan-out, time bounds, atomic state.
* [`.theloop/RULE-FILES.md`](.theloop/RULE-FILES.md) — directory rules: plain-English gates judged over scoped files, registered in [`ai-rules.yml`](ai-rules.yml).
* [`.theloop/CACHING.md`](.theloop/CACHING.md) — content-fingerprint caching of agentic checks.
* [`PRECOMMIT.md`](PRECOMMIT.md) — the repo-specific checks the pre-commit gate runs on top of rules and skill validation.
* [`SKILLS.md`](SKILLS.md) and [`.theloop/VIZ.md`](.theloop/VIZ.md) — the exact inventory and topology of the skills in this repo.

`AGENTS.md` is a symlink to this file: a coding agent reading the standard entry point gets the same principles, from the same bytes.

## 1. One source of truth — aliases are links, copies are bugs

`theloopify` is a symlink to `theloopify.sh`, not a copy. Each maxims category is a `.yml` source of truth with its `.md` generated byte-identically from it. `ai-rules.yml` exists so no run ever crawls the repo for rules — and it must match the repo exactly, in both directions. `SKILLS.md` and `.theloop/VIZ.md` are required to mirror the repository precisely. The recurring failure mode in this repo's history is two artifacts drifting apart; the recurring fix is making one of them mechanical. When you are about to copy something, link it or generate it instead.

## 2. Names carry contracts

Every skill name is lowercase-dash-separated. The `theloop-internal-` prefix means — mechanically enforced, so the name cannot lie — that the skill takes a caller-supplied `SkillRunId` and is never typed by a human. User-facing entry points get short, discoverable names. A development name differs from its client-facing install name only when collision forces it (`theloop-clientrepo-precommit` installs as `theloop-precommit`), and the installer renames it so the client never sees the seam.

## 3. Mechanical before agentic

Anything a script can check, a script checks: single-shot Python primitives under each skill's `scripts/`, executed directly by path, never improvised by the runner. Agentic judgment is reserved for what scripts cannot express — and even then it is cached by content fingerprint per [`.theloop/CACHING.md`](.theloop/CACHING.md). Parallelism lives in the skill prompt (fan out subagents), never inside the Python.

## 4. Every state mutation is atomic, idempotent, and lock-free

Write to a temp file and atomically rename, so a file's existence is proof of completed work. Use optimistic concurrency with `RunId` nonces, never a blocking lock: the skills are driven by an agentic runner whose steps may stall, retry, or never resume, and a lock held by a hung runner deadlocks. We accept a small, self-healing chance of a lost update over the risk of a deadlock — deliberately.

## 5. Nothing waits forever

Every operation a skill awaits — a spawned sub-run, an invoked script, an external `gh` or `git` call — is bounded by the caller and ends as either completion or a reported timeout error, never a hang. A sub-run that never wrote its receipt by its deadline *is* a timeout error, not a reason to keep waiting.

## 6. Split slow collection from cheap analysis, and cache the collection

When work is slow, collect each input's data into a file named by its identity, then analyze from the cache. Mark an input processed only after its output is durable. Cache passes only — a failure is never cached. The payoff is crash-safe resumption: after an interruption, at worst repeat cheap analysis; never re-fetch, never lose an input, never journal one twice.

## 7. The gate gates everything

No commit lands without a passing `theloop-precommit` run, and the commit message says so. The gate is treated as part of the product: exercise it, extend it through [`PRECOMMIT.md`](PRECOMMIT.md), and when it fails, let it drive the fix.

## 8. Document the compromise, not just the rule

When the repo breaks its own principle, it says so explicitly, states the cost, and contains the blast radius — `theloopify` modifying a client's `.gitignore` is "a deliberate compromise", documented as such, with the change excluded from commits. Undocumented exceptions are the only forbidden kind.

## 9. Reviewer-friendly history

Split work into logical, separable commits. Reference `Issue #N` and `PR #N` in the standard form. Journal progress on the issue as it happens — every push, every gate failure — not reconstructed after the fact.
